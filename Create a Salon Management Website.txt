Create a Salon Management Website :
:
Aim:
      To develop a Membership Tracking System for A Salon to improve Loyalty. A Salon Owner wants to manage customers and wants to provide different service packs to the customers.

What is expected from the website:

1. Users Section: 
   a) There will be one SUPER USER, viz the developer
   b) The Super User will create accounts of different Salons. Super User and Each Salon Owner will be allotted User no, which will not be  user name or user id. It will be a running serial number.
   c) Salon Owner will login from Users menu. Each Salon Owner will create accounts for  Salon Manager iii) different Customers. There can be multiple Salon Managers.
   d) Users menu will have three options of logins, one each for Salon Owner, Salon Manager iii)  Customers.
   e)  Another additional menu option for login will be "ADM User".  Under "ADM User" option there will be provision for add/modify/delete/activate/deactivate/archive  a Customer or Salon Manager
   f) For any User (Except SUPER USER) , there will be fields for Name, Sex, Password, Phone number, email-id, Address, City, Google map link(Optional), age-group, distance.
   g) Age groups will be Kids (3–12), Teens (13–19), Young Adults (20–29), Professionals (30–45), Gen X (Ages 46–60), Seniors (61+)
   h) Distance to be accepted from the options : Hyper-Local Range (0–3 km), Local Range (3–7 km), Mid Range(8-11 km), Regional Range (12+  km)
   i) A Customer Number will be generated on creation of Customer. It will be first three letters of the name and a three digit running serial number.
   j) There is no Customer login at present. Password is null for the Customer. 
   k) Each customer will be allotted a simple Customer Number. Customer Number will be First Three Letters of Customer Name and a running three-digit serial number with zero padding on the left. 
  l)  User details be shared by email to the user creating it  and , whenever  the user is created or modified or deleted or archived.
  m) Email-id be OTP verified
  n) A Salon Owner should be able to delete/activate/deactivate/archive only Salon Manager Users  and Customer users created by the Salon Owner himself (and not other Salon Owners), whereas Super User can delete/activate/deactivate/archive any Salon Owner User or Salon Manager User or Salon's customers.
  o) When a Super User  deletes/activates/deactivates  Salon Owner  user then  Salon Manager and all Customer user under this Salon Owner user are  deleted/activated/deactivated.  
  p) While creating a Salon Owner user, Salon Owner's user folder name be accepted in the UI and this  user folder name will be stored in User's record . Some of the files will be stored in this user folder and will be part of data for that user.
 q) Address field should be multiline text box
 r) During displays of data, Contact Numbers to be clickable and on click should be copied to clipboard, Google map link to be clickable and Google map should be opened on clicking.
 s) Cascade delete option be considered, whereever necessary  but before delete confirmation be made from the user with WARNING beep and Message.
 t) A Parent User field will be part of User record, which will be User no of the parent.
 u) There will be timeout of 15 minutes for logins.
 v) SUPER USER login option will not be shown after Salon Owner/ Manager login .
 w) SUPER USER login will be through emailid and OTP option and not through password option.
 x) When a Salon Owner/Salon Manager is to be logged in, following checks will be required for User :
     i) current date to be more than startDate
    ii) Either expiryDate is null or current date is not more than expiryDate 
   iii) User is not deactivated.

9. Common Service Packs Section:
   a) There will be Seasonal Discounts, Discounts on Special Occasions like New Year, Christmas, Diwali, Marriage Season, Navratre, Independence/Republic Day, Bhaiya Dooj, Rakhi, Karva Chauth. Each of the above discounts will have Start and Expiry Date. Expiry date will be optional. Start and Expiry Date may be accepted from the Salon Owner/Manager. 
   b) There will be different service packs 
   d) Each Service Pack will have a Title, a Total Cost, overall discount and Offer Price.
   c) Each Service Pack will have various Sub-Services, their actual rates, discounts
   d) Common Service Pack Id will be First Three Letters of Service Pack Name/Title and a running two digit number.
   e) Discount Code will be First Three Letters of Discount Name/Title and a running two digit number.
  g) Duration to serve the service pack will be sum of durations of each sub-service under it. It will be editable field to account for some sub-services being imparted simultaneously.
  h) Time, in minutes,  to be taken to serve the customer (durationMinutes) will not be part of  Service Category or Service entities.
  i) Offer Price of the Service Pack will be sum of the Final Prices of each of the sub-services under it..
  j) Total Cost of the Service Pack will be sum of the rates of each of the sub-service under it.
  k) Overall Discount of the Service Pack will be back calculated from above-mentioned Offer Price and Total Cost 
   l) There will be option of Sharing the common service packs available so far by  the Salon Owner/Manager  
   m) An flyer image file for a service pack for promotional purposes be accepted from the Salon Owner PC/mobile.
  n) An option for the error log be there in the menu system. Technical Errors in detail should be displayed and recorded in an error log file in the same folder as the index.html file or app file.

13.  Membership deals /Service Pack Allotment to a Customer by Salon Owner/Manager
  a) User Interface (UI) will offer a Customized Service Pack Creation screen/sub screen 
  b) Customer number will be accepted then UI will  display service packs available and their details 
  c) Then it will display options for Add/Modify/Delete/ Activate/Deactivate/Archive options for Membership deals
  d) In case of creation of Service Pack, It will generate a name for the Service Pack to be Created. The name for the service pack will  be like "SP-<n>", "n" here will be running serial number, which will start from 1 for each different customer.
  e) For creation/modification of service pack, in the UI, Sub-Services will be added to/deleted from this pack. UI wiil accept alongwith discount offered for the said service,  allowed number of services of the said service, active/inactive (active initally), coins to be imparted to the customer (if any) when the said service is availed . In case any number of services of a particular sub-service are allowed, a check box be provided for that and if this flag is yes, counter for  number of services allowed be set to zero .
  f) Net amount, after discount, rounded to nearest tens figure will be shown. This Net amount should be editable for negotiations.   
  g)  For each different Customer, running serial numbers will start with 1.
  h) Start date of a service pack will be current date and editable for making past entries.
  i) Expiry date, if any, and initally null, of the service pack be accepted.
  j) A running total amount and offer price  be shown which will be updated after addition of every sub-service.
  k) As customer may negotiate a offer price. The offer price will be editable and saved alongwith.
  l) There will be two types of service packs to be called Type 1 and Type 2. Service Pack Types (packType field)  will be saved alongwith the service packs info.
  m) In Type 1, sub-services will be added and their discounts. After creation of pack, An optional payment to be accepted alongwith receipt number, if any, and date.
  n) In Type 2, After creation of pack, an advance will be accepted and overall discount in the services will be offered. This discount will be applied to all the services under this service pack.  An payment to be accepted alongwith receipt number, if any and date.
o) Customer Service pack may be for Self/Spouse/Couple/Family/Extended Family. Option be taken from customer.
p) Provision for Modification/Deletion/View/Archive of Record by Owner only.
q)  An entry will be made in database, whether data was entered by Owner or Manager.
r) For  the new customer service pack, there will be additional field per sub-service  in database for no of coins earned on availing a sub-service after the sub-service is served. total coins earned for this service pack will have  initial value 0. 
s) Duration to serve this service pack will be sum of durations of each sub-service under it. It will be editable field to account for some sub-services being imparted simultaneously.
t) Upon Allotment of a Service Pack to a customer after creation of the record an email will be sent to the Salon Owner alongwith the details of the service pack allotted.
u) There should be option of Sharing the service pack with the customer,  containing all the fields of the service pack. Option of Email/Text Message/Whatsapp be accepted from Customer. In case of Whatsapp, a flyer be sent.
v) There will be two sub-schemas for customer service pack  i) For service pack containing fields like name, start date, expiry date, total coins earned , active or inactive, service pack  type, total amount, amount received or not, receipt no (optional), receipt date  ii) For sub-services selected under the service pack containing fields like sub-service, price, discount, coins to be imparted for the sub-service, coins earned upon availing the service, date of availing sub-service,  data entered by Owner or Manager,  number of sub-services proposed, check box for any number of sub-services proposed (i.e. no limit on number of sub-services)

15. Utilization of Service Pack by Customer (Entries allowed by Salon Owner and Manager Both)
    a) When a customer visits Salon and utilizes a service pack allotted to him/her, open a screen accepting customer number..
    b) Customer number will be accepted then UI will present screens for ii) Utlilization of  membership services ii) Display of service packs details
    c) Customer Service pack Name to  be accepted. For Display of service packs details, UI may also accept option for displaying details of all service packs of this customer.
